# packages/audiodocs/ — Documentation Site

Docusaurus 3 static site. Docs are MDX files in `docs/`. Custom React components live in `src/components/`.

## Commands

```bash
yarn start   # dev server with hot reload
yarn build   # production build
yarn serve   # serve the built site
```

## Directory Structure

```
packages/audiodocs/
├── docs/                  # All documentation pages (MDX)
│   ├── fundamentals/      # Getting started, installation, best practices
│   ├── guides/            # Tutorials (piano keyboard, noise, ADSR envelope)
│   ├── core/              # AudioContext, AudioNode, AudioParam, BaseAudioContext
│   ├── sources/           # Oscillator, BufferSource, ConstantSource, MediaElement
│   ├── effects/           # Gain, BiquadFilter, Convolver, Dynamics, StereoPanner, Delay, IIR...
│   ├── analysis/          # AnalyserNode
│   ├── inputs/            # AudioRecorder
│   ├── destinations/      # AudioDestinationNode
│   ├── system/            # AudioManager, PlaybackNotificationManager
│   ├── worklets/          # AudioWorklet setup and nodes
│   ├── types/             # Enum docs (OscillatorType, BiquadFilterType...)
│   ├── utils/             # Utility functions (decoding, time-stretching)
│   ├── react/             # React-specific hooks and helpers
│   └── experimental/      # Experimental APIs (Audio tag / development.react)
├── src/
│   ├── components/        # Custom MDX components
│   ├── examples/          # Standalone interactive demo implementations
│   └── pages/             # Landing page
├── static/                # Images, audio files
├── sidebars.js            # Autogenerated — no manual edits needed
└── docusaurus.config.js   # Site config
```

## Sidebar / Navigation

Sidebar is **fully autogenerated** from the folder structure — no edits to `sidebars.js` needed.

Order is controlled by:
- **Between folders**: `_category_.json` → `position` field
- **Within a folder**: frontmatter `sidebar_position: N` in each `.mdx` file

To add a new category folder, create `_category_.json`:
```json
{ "label": "My Category", "position": 6 }
```

## MDX File Format

All doc pages are `.mdx` files with YAML frontmatter.

### Minimal frontmatter

```yaml
---
sidebar_position: 3
---
```

### Full page structure for an audio node

```mdx
---
sidebar_position: 3
---

import AudioNodePropsTable from "@site/src/components/AudioNodePropsTable"
import { Optional, ReadOnly } from '@site/src/components/Badges';

# GainNode

Brief one-line description. `GainNode` extends [`AudioNode`](/docs/core/audio-node).

#### [`AudioNode`](/docs/core/audio-node#properties) properties

<AudioNodePropsTable
  numberOfInputs={1}
  numberOfOutputs={1}
  channelCount={2}
  channelCountMode={"max"}
  channelInterpretation={"speakers"}
/>

## Constructor

```tsx
constructor(context: BaseAudioContext, options?: GainOptions)
```

### `GainOptions`

Inherits all properties from [`AudioNodeOptions`](/docs/core/audio-node#audionodeoptions).

| Parameter | Type | Default | |
| :---: | :---: | :----: | :---- |
| `gain` <Optional /> | `number` | `1` | Initial gain value. |

Or by using [`BaseAudioContext.createGain()`](/docs/core/base-audio-context#creategain).

## Example

```tsx
import { AudioContext } from 'react-native-audio-api';

const ctx = new AudioContext();
const gain = ctx.createGain();
gain.gain.value = 0.5;
gain.connect(ctx.destination);
```

## Properties

It inherits all properties from [`AudioNode`](/docs/core/audio-node#properties).

| Name | Type | Default | Description |
| :----: | :----: | :-------- | :------- |
| `gain` <ReadOnly /> | `AudioParam` | `1` | Gain applied to the signal. Range: [0, 1] typical. |

## Methods

It inherits all methods from [`AudioNode`](/docs/core/audio-node#methods).

## Remarks

#### `gain`
- Nominal range is [0, 1] for attenuation but accepts any finite value.
- Values > 1 amplify; negative values invert phase.
```

### InteractivePlayground — live audio demo with controls

Use when you have a `useXxxPlayground` hook in `src/components/InteractivePlayground/`.

```tsx
import InteractivePlayground from '@site/src/components/InteractivePlayground';
import { useOscillatorPlayground } from
  '@site/src/components/InteractivePlayground/OscillatorExample/useOscilatorPlayground';

<details open>
<summary><code>OscillatorNode</code> interactive playground</summary>
<InteractivePlayground usePlayground={useOscillatorPlayground} />
</details>
```

To add a new playground: create `src/components/InteractivePlayground/MyExample/useMyPlayground.ts`.

### InteractiveExample — side-by-side code + rendered output

## Adding a New Node Doc Page

1. Identify the category folder: `docs/sources/`, `docs/effects/`, `docs/analysis/`, etc.
2. Create `docs/<category>/my-node.mdx`.
3. Use the structure above: frontmatter → imports → `AudioNodePropsTable` → Constructor → Example → Properties → Methods → Remarks.
4. Set `sidebar_position` relative to neighboring files (check their positions first).
5. Sidebar updates automatically on next `yarn start` / `yarn build`.

## Adding a New Guide / Tutorial

1. Create `docs/guides/my-guide.mdx`.
2. Use `<InteractiveExample>` to embed working code + rendered output.
3. Use `<InteractivePlayground>` for live parameter control.
4. Add the example code in `src/examples/MyGuide/`.

## Property Table Format

```mdx
| Name | Type | Default value | Description |
| :----: | :----: | :-------- | :------- |
| `frequency` <ReadOnly /> | `AudioParam` | `440` | Oscillation frequency in Hz. |
| `type` | `OscillatorType` | `'sine'` | Waveform shape. |
```

`<ReadOnly />` — property cannot be reassigned (AudioParam wrappers are always ReadOnly).
`<Optional />` — constructor option or parameter is optional.
`<IOS />` / `<Android />` — feature is platform-specific.
